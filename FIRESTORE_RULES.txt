Go to Firebase Console > Firestore Database > Rules
Replace with this:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /clips/{clipId} {
      // Allow queries - the where('userId', '==', userId) clause filters to user's own clips
      allow list: if request.auth != null;
      
      // Allow reading individual document if user owns it
      allow get: if request.auth != null 
        && request.auth.uid == resource.data.userId;
      
      // Allow creating clips with own userId
      allow create: if request.auth != null 
        && request.auth.uid == request.resource.data.userId;
      
      // Allow updating own clips
      allow update: if request.auth != null 
        && request.auth.uid == resource.data.userId;
      
      // Allow deleting own clips
      allow delete: if request.auth != null 
        && request.auth.uid == resource.data.userId;
    }
  }
}

Then click "Publish"

IMPORTANT: The 'list' permission allows queries. Security is maintained because:
- The app always queries with where('userId', '==', userId)
- Firestore enforces the where clause server-side
- Users can only see their own clips even with list permission
